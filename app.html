Framework Core Files
  • index.html : It is used as a container, only add meta info and dependencies to this file
  • app.html : Write the usual markup here, this is automatically injected into index.html
  • mainfest.json : Provides app info and skyroute definitions to the Browser
  • sw.js : Our service worker file, SkyRoutes takes care of everything here, only to push a force upgrade, edit the 'force_upgrade_string'
  • skyroutes.js, skyroutes.css : SkyRoutes core files
Note
  • The location of the core files is crucial for the framework to function
Initializing our app We need to initialize a skyroutes object in the mainfest.json file
Key Value Description
assets Array of Valid paths as String This is used for specifing files to be cached by the PWA, you can edit this string anytime
hot_reload css/silent/full/false refresh css / refresh html and css / reload app / disabled based on focus change
force_install true/false Force user to add app to HomeScreen
force_network true/false App dependent on active internet connection
route_error hold/free Client grabs navigation upon incorrect route (path) / usual server fetching
update_interval -1/0/int in minutes atomic updates disabled / atomic updates every reload / atomic updates after duration (in minutes)
Note: pathEvents passes the current route which can be used in some way by the app logic
