Sample App Structure

App Core
  • index.html is used as a container, only add meta info and dependencies to this file
  • app.html is where for the usual markup goes, this is automatically injected into index.html
  • main.js is used as the entry point for JavaScript
  • mainfest.json provides PWA's info to the Browser, update this file accordingly
  • sw.js is our service worker file, SkyRoutes takes care of everything here, only to push a force upgrade edit the force_upgrade_string
Note
  • The location of the core files is crucial for the framework to function
  • In the current release SkyRoutes is designed to only work under a root domain (example.com), it will not work with subdomains (example.com/pwa)
Initializing our app We need to initialize a myApp Object in our main.js file
Key Value Description
name Valid or empty String App name for splash screen, on empty defaults to 'SkyRoutes'
author String For reference
logo Valid path as String or empty App logo for splash screen, on empty deafults to skyroutes logo
assets Array of Valid paths as String This is used for specifing files to be cached by the PWA, you can edit this string anytime, on empty array the whole app cache is erased
hot_reload css/silent/full/false refresh css / refresh html and css / reload app / disabled based on focus change
force_install true/false Force user to add app to HomeScreen
force_network true/false App dependent on active internet connection
route_error hold/free Client grabs navigation upon incorrect route (path) / usual server fetching
update_interval -1/0/int in minutes atomic updates disabled / atomic updates every reload / atomic updates after duration (in minutes)
anti_scrape true/false (WIP) Halt app if being web scraped
change String For internal reference
Note: pathEvents passes the current route which can be used in some way by the app logic
